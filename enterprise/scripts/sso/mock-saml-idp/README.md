# Mock SAML IdP Fixture

Local-only SAML 2.0 Identity Provider used for integration tests and manual SAML flow walkthroughs in `web-portal` / `admin-console`.

> The keypair generated by `setup-keypair.sh` is **not** for production. Never copy it to a deployed environment.

## Usage

```bash
# 1. Generate the keypair / self-signed certificate (once per checkout)
bash enterprise/scripts/sso/mock-saml-idp/setup-keypair.sh

# 2. Start the fixture (defaults to 127.0.0.1:4444)
node enterprise/scripts/sso/mock-saml-idp/index.mjs

# Optional: override the host / port / entityID
MOCK_SAML_IDP_PORT=4444 \
MOCK_SAML_IDP_HOST=127.0.0.1 \
MOCK_SAML_IDP_ENTITY_ID="http://127.0.0.1:4444/" \
node enterprise/scripts/sso/mock-saml-idp/index.mjs
```

## Endpoints

- `GET /metadata` — IdP metadata XML, contains the signing X.509 certificate and SSO binding URLs.
- `GET /sso?acs=<acsUrl>&audience=<spEntityId>&email=<email>&displayName=<name>&roles=<csv>&RelayState=<state>&inResponseTo=<requestId>` — Returns an auto-submitting HTML form that POSTs a signed SAMLResponse to the SP's ACS URL. `acs` and `audience` are required.

## Configuration tips

- Configure the matching SAML provider in admin-console (`/settings/sso`) with:
  - `idpEntityId` = `http://127.0.0.1:4444/`
  - `idpSsoUrl` = `http://127.0.0.1:4444/sso`
  - `idpCertPemList` = contents of `.fixture/idp-cert.pem`
  - `spEntityId` = your SP entity id (e.g. `http://localhost:3000/`)
  - `acsUrl` = `http://localhost:3000/api/auth/sso/saml/callback` (portal) or `http://localhost:3001/api/auth/sso/saml/callback` (admin-console)
- The fixture intentionally signs only the `Assertion` element (matching `wantAssertionsSigned: true`, `wantResponseSigned: false`).
