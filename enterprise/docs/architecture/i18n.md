# Enterprise 国际化（i18n）

admin-console（`:3001`）与 web-portal（`:3000`）共用 **next-intl** + **cookie 驱动 locale**，默认 `zh`，可选 `en`。

## Locale 存储

| 机制 | 键名 | 说明 |
|------|------|------|
| Cookie（SSR 主来源） | `NEXT_LOCALE` | 值 `zh` 或 `en`；`src/i18n/request.ts` 的 `getRequestConfig` 读取 |
| localStorage（兼容） | `agenticx-ui-locale` | `setLocale()` 双写，兼容旧会话 |

切换语言：顶栏/用户菜单 → 中文 / English → 写 cookie 并 `router.refresh()`，首屏与 SSR 一致。

## 目录结构

```
apps/{admin-console,web-portal}/
├── messages/
│   ├── zh.json
│   └── en.json
├── src/i18n/
│   ├── routing.ts    # locales、LOCALE_COOKIE
│   └── request.ts    # getRequestConfig → 从 cookie 加载 messages
```

**不使用** `src/middleware.ts`（无 `/en` 前缀的 cookie 方案见 [next-intl without i18n routing](https://next-intl.dev/docs/getting-started/app-router/without-i18n-routing)；middleware 会对 `/` 等路径做内部 rewrite 导致 404）。

`@agenticx/ui` 的 `LocaleProvider` / `useLocale()` 为薄壳，不依赖 next-intl，由 app 注入 `initialLocale` 与 `onLocaleChange`。

## 命名空间约定

| 前缀 | 用途 |
|------|------|
| `shell.*` | AppShell：导航、命令面板、主题/语言菜单 |
| `common.*` | 跨页：保存/取消/加载/启用禁用 |
| `pages.*` | 按路由：如 `pages.admin.perf.title` |
| `portal.*` / `workspace.*` / `chat.*` / `settings.*` | web-portal 各区域 |

新增页面时：**先在 `messages/zh.json` 与 `en.json` 成对添加键**，再在组件内 `useTranslations('pages.xxx')`。

## 组件内用法

**Client Component：**

```tsx
import { useTranslations } from "next-intl";

export function MyPage() {
  const t = useTranslations("pages.admin.example");
  return <PageHeader title={t("title")} description={t("description")} />;
}
```

**Server Component：**

```tsx
import { getTranslations } from "next-intl/server";

export default async function Page() {
  const t = await getTranslations("pages.admin.example");
  return <h1>{t("title")}</h1>;
}
```

**与 `useLocale()` 并存：** 仅需切换语言时用 `useLocale().setLocale('en')`；文案一律走 `useTranslations`，不要手抄 `SHELL_COPY` 字典。

## 本地验证

```bash
# 设 cookie 后强制刷新，<html lang> 应为 en
document.cookie = "NEXT_LOCALE=en;path=/;max-age=31536000"
location.reload()
```

## ESLint

`apps/admin-console` 与 `apps/web-portal` 的 `src/**` 启用 `eslint-plugin-i18next` 的 `no-literal-string`（排除测试与注释）。PR 中新增用户可见中文/英文长句会 lint 失败。

允许列表见各 app 的 `eslint.config.mjs`（品牌名 Machi、AgenticX、HTTP 状态码等）。

## 非目标

- 无 `/en/...` URL 前缀（`localePrefix: "never"`）
- 后端 API `message` 字段不做 i18n（前端可按 code 映射，未命中则原文展示）
- `@agenticx/ui` 原语组件不接 next-intl，由调用方传入已翻译字符串
